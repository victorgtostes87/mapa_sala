Como usar o sistema

Guia rápido para consulta no dia a dia. O sistema é um apoio operacional e não substitui o prontuário nem o sistema clínico oficial.

Recepção Coordenação Restaurar backup
{% if topico == 'recepcao' %}

Como usar: recepção

  1. Abra o Painel para ver pendências do turno.
  2. Use Horários abertos para encontrar alunos que ainda podem receber paciente ou triagem.
  3. Use Reservas para aprovar ou recusar pedidos de sala e acompanhar instrumentos.
  4. Use Afazeres como lista compartilhada, no lugar de post-its no monitor.
  5. No mapa, cadastre somente informações operacionais: aluno, sala, horário, categoria e se é triagem.
Evite colocar dados clínicos sensíveis em observações. Se a informação pertence ao atendimento, ela deve ficar no sistema clínico oficial.
{% elif topico == 'coordenacao' %}

Como usar: coordenação

  1. Use o Painel da coordenação para acompanhar alunos sem paciente, triagens abertas e horários ociosos.
  2. Use Relatório semanal para uma visão resumida do uso das salas e supervisões.
  3. Use Usuários para cadastrar, editar, inativar ou excluir cadastros criados por engano.
  4. Use Logs para conferir ações importantes: login, criação, edição, exclusão, importação, backup e apagamento de logs.
  5. Use Backup antes de importar Excel ou fazer mudanças grandes no mapa.
Permissões críticas ficam com coordenador: excluir usuário, excluir agendamento, importar Excel, baixar backup e apagar logs.
{% else %}

Como restaurar backup

  1. Avise a coordenação e combine um horário sem usuários usando o sistema.
  2. Baixe o backup mais seguro disponível e guarde uma cópia do banco atual antes de substituir qualquer arquivo.
  3. No PythonAnywhere, abra o console Bash e entre na pasta do projeto.
  4. Substitua o arquivo do banco SQLite pelo backup escolhido.
  5. Rode o comando de manutenção e depois faça reload do Web App.
  6. Entre no sistema e confira usuários, mapa, reservas e logs principais.
cd ~/mapa_sala
flask --app app manutencao
flask --app app manutencao --vacuum
Restaurar backup troca o estado do sistema. Faça isso só quando houver erro real, importação incorreta ou perda de dados.
{% endif %}
